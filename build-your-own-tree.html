Build Your Own Tree

This page walks through the complete analysis pipeline used to generate the trees on the previous page — from downloading raw sequences off GenBank to producing a publication-ready PDF figure in R. All four R scripts are available in the project GitHub repository and can be run on any laptop running R 4.0 or higher. The guide is written for upper-division undergraduates and beginning graduate students who are comfortable in R but new to phylogenetics.

What You Need

You will need R version 4.0 or higher, which is available at r-project.org. The analysis uses four R packages: ape and phangorn are on CRAN and can be installed in the standard way; msa and Biostrings are Bioconductor packages and require the BiocManager installer. Run the following block once in your R console to get everything in place, then leave the lines commented out in your scripts so they don't re-install on every run. 

install.packages(c("ape", "phangorn"))
if (!requireNamespace("BiocManager")) install.packages("BiocManager")
BiocManager::install(c("msa", "Biostrings"))

For the Bayesian analysis you also need MrBayes, which runs outside R from the terminal. On a Mac with Homebrew installed, brew install mrbayes handles the whole thing. On other systems, precompiled binaries and source code are available at mrbayes.sourceforge.net. Once installed, confirm it works by typing mb in a terminal window — you should see the MrBayes welcome banner.

All four R scripts for this project (00_prepare_alignment.R, 01_tree_NJ.R, 02_tree_ML.R, 03_tree_Bayesian.R) are in the R/ folder of the project GitHub repository. Clone the repository or download the scripts individually and place them in a folder called R/. The scripts expect to be run from that folder as the working directory, and they will create out/rdata/ and out/trees/ subdirectories automatically on first run.

Getting Sequences from GenBank

All sequences in this analysis come from NCBI Nucleotide, the public repository for DNA sequence data. The marker we used is COI — cytochrome oxidase subunit I, a mitochondrial protein-coding gene that has become the standard barcode for animal species identification. COI is ideal for this kind of analysis because it sits in a sweet spot: it is conserved enough across the animal kingdom that a single pair of universal PCR primers can amplify it from almost any fish species, but it is variable enough between species that it reliably distinguishes them from one another. The standard barcode region is approximately 650 base pairs long.

To find COI sequences for a new species, go to ncbi.nlm.nih.gov/nucleotide and search using the query [species name][ORGN] AND COI[GENE] AND 600:700[SLEN]. The [ORGN] field filters by organism name, [GENE] restricts to sequences annotated as COI, and 600:700[SLEN] limits results to the barcode length range, excluding very short fragments and whole mitogenomes. Choose two or three sequences per species if available — having replicates lets you verify consistency, as described on the previous page. Download results in FASTA format. Once you have individual FASTA files, combine them into one file from the terminal with cat *.fasta > combined.fasta, then open the file in a text editor to give each sequence a clean, short header like >Red_Drum_1 before passing it to R.

Script 00: Alignment and Setup

00_prepare_alignment.R is the shared setup script that all three tree scripts depend on. Run it once — it takes about 30 seconds — and it handles everything from reading raw sequences to writing the NEXUS file for MrBayes.

The script opens the combined FASTA using Biostrings::readDNAStringSet() rather than ape's read.dna(). This matters because MUSCLE (the aligner) expects Biostrings format as input, and going via ape's DNAbin format and then converting can introduce encoding artifacts for ambiguous base codes — a silent error that produces a cryptic message about unexpected characters. Starting in Biostrings format and converting after alignment cleanly avoids this.

Before any tree can be built, every sequence must be aligned — that is, gaps must be inserted so that column n in the alignment matrix contains the same homologous nucleotide position across all 24 sequences. This step is mandatory because distance calculations and likelihood functions both assume you are comparing the same site in each sequence. Without alignment, you would be comparing random positions to each other. The MUSCLE algorithm handles this in seconds for a dataset this size, producing an aligned matrix of approximately 660 columns. 

# Read unaligned sequences directly into Biostrings format
dna_strings <- Biostrings::readDNAStringSet(
  "FASTA/gulf_fish_COI_combined.fasta"
)

# Align with MUSCLE
msa_result <- msa(dna_strings, method = "Muscle")
cat("Alignment length:", ncol(as.matrix(msa_result)), "bp\n")

# Convert to ape (for NJ distances) and phangorn (for ML)
aln_dnabin <- msaConvert(msa_result, type = "ape::DNAbin")
aln_phydat <- msaConvert(msa_result, type = "phangorn::phyDat")

# Save both objects plus the color helper for downstream scripts
save(aln_dnabin, aln_phydat, family_colors, tip_color,
     file = "out/rdata/alignment_objects.RData")

The script also writes two output files: FASTA/gulf_fish_COI_aligned.fasta for inspection in alignment viewers like AliView or MEGA, and MrBayes/gulf_fish_COI_aligned.nex as the NEXUS data block that 03_tree_Bayesian.R will append the MrBayes analysis settings to. The alignment_objects.RData file is the main handoff to all three tree scripts — it contains the aligned matrix in two formats plus the shared color palette used by all figures.

Script 01: Neighbor-Joining

01_tree_NJ.R builds two NJ trees — one rooted on Bull Shark and one rooted on Flathead Mullet — and saves them with 1000 bootstrap replicates each. NJ is the fastest of the three methods and serves both as a quick sanity check on the data and as the starting topology for ML optimization.

The first step is computing pairwise distances under the K80 model (Kimura 2-parameter, 1980). Raw p-distance — the fraction of positions that differ between two sequences — underestimates true evolutionary distance because the same site can change more than once, with later changes erasing the evidence of earlier ones. K80 corrects for this multiple-hit problem. It also accounts for transition/transversion bias: in mitochondrial DNA, transitions (purine-to-purine: A↔G; or pyrimidine-to-pyrimidine: C↔T) happen two to three times more often than transversions (purine-to-pyrimidine). K80 estimates a separate rate for each category, which makes it significantly more accurate than p-distance while requiring only two parameters.

The NJ algorithm itself (Saitou & Nei 1987) builds the tree by iteratively joining the pair of taxa whose merger minimizes the total branch length of the tree — the minimum evolution criterion. It is computationally fast (O(n²) for n taxa) and gives a good approximation of the true topology in most cases, though it can struggle when rate variation among lineages is severe. 

# Compute K80 pairwise distances
dist_k80 <- dist.dna(aln_dnabin, model = "K80")

# Build unrooted NJ tree, then root on Bull Shark
tree_nj_raw   <- nj(dist_k80)
outgrp_shark  <- grep("Bull_Shark", tree_nj_raw$tip.label, value = TRUE)
tree_nj_shark <- root(tree_nj_raw, outgroup = outgrp_shark,
                       resolve.root = TRUE)

# Bootstrap: resample columns 1000 times, rebuild NJ each time
boot_nj_shark <- boot.phylo(
  tree_nj_shark, aln_dnabin,
  function(x) root(
    nj(dist.dna(x, model = "K80")),
    outgroup = grep("Bull_Shark", rownames(x), value = TRUE),
    resolve.root = TRUE
  ),
  B = 1000, quiet = TRUE
)

When reading the output PDF, look for three things: whether all members of the same species cluster together (validates the sequence labels), whether all four Sciaenidae cluster together to the exclusion of other families (tests family monophyly), and the relative branch lengths. A very long branch on the outgroup and short branches among closely related ingroup species is the expected pattern for a working barcode dataset. Bootstrap values at the node uniting all Sciaenidae should be at or near 100%; values below 70% at internal nodes within the family indicate genuine uncertainty about branching order rather than errors in the data.

Script 02: Maximum Likelihood

02_tree_ML.R runs model selection, fits a GTR+Gamma+I ML tree starting from the NJ topology, runs 1000 bootstrap replicates, and saves the result. Plan for 10–20 minutes of compute time on a standard laptop for this dataset; larger datasets scale roughly linearly with the number of taxa.

The script first runs modelTest() from phangorn, which fits dozens of standard substitution models to the data and ranks them by AIC (Akaike Information Criterion). Lower AIC means better fit penalized for model complexity. Regardless of the winner, the script proceeds with GTR+Gamma+I — the most general standard model — as a conservative and widely accepted choice for publication. GTR (General Time Reversible) allows all six possible substitution rates to differ from one another, rather than assuming any symmetry between them. The +Gamma component adds a shape parameter α that controls how much substitution rate varies across the alignment: a small α means a few positions are evolving very fast while most are nearly frozen, while a large α means rates are relatively uniform. The +I component estimates the fraction of sites that are completely invariant — a biologically realistic feature of conserved protein-coding genes like COI. 

# Start ML search from the NJ topology (faster convergence)
fit_start <- pml(tree_nj_start, data = aln_phydat)
fit_ml    <- optim.pml(
  fit_start,
  model         = "GTR",
  optGamma      = TRUE,        # estimate Gamma shape parameter
  optInv        = TRUE,        # estimate proportion invariant
  rearrangement = "stochastic",# thorough topology search
  control       = pml.control(trace = 0)
)

# 1000 bootstrap replicates (~10-20 min)
bs_ml      <- bootstrap.pml(fit_ml, bs = 1000, optNni = TRUE,
                              control = pml.control(trace = 0))
tree_ml_bs <- plotBS(fit_ml$tree, bs_ml, type = "n")

The rearrangement = "stochastic" option combines random nearest-neighbor interchange moves with standard NNI, making the topology search more thorough than plain NNI at the cost of extra time. Starting from the NJ tree rather than a random topology reduces the risk of the optimizer settling into a poor local maximum.

Bootstrap values are interpreted on a standard scale: values at or above 95% are considered strong support and can be reported with confidence; 70–94% indicates moderate support, worth noting with a brief caveat; 50–69% is weak and should be described as tentative; values below 50% are typically not reported in published trees and indicate that the branching order at that node is not recoverable from this dataset. For a single 650 bp barcode gene, it is normal and expected to see weak bootstrap support at internal nodes within a recently diversified family like Sciaenidae.

Script 03: Bayesian Inference

03_tree_Bayesian.R uses a two-step workflow. On the first run, the script checks that the NEXUS file exists, appends the MrBayes analysis block to it, prints instructions, and exits. You then run MrBayes from the terminal. On the second run, the script detects the MrBayes output, reads the consensus tree, and produces the figure.

What MrBayes is doing

Bayesian inference uses Markov chain Monte Carlo (MCMC) sampling to explore tree space. Rather than finding a single best tree (as ML does), it runs a chain that randomly proposes new trees, accepts or rejects each proposal based on how much it increases or decreases the posterior probability of the data, and records every accepted tree. After enough generations, the set of sampled trees approximates the full posterior distribution — a probability distribution over all possible trees given the data and the model. The first portion of the run, called the burn-in, is discarded because the chain hasn't yet converged to the stationary distribution. In this analysis we ran 500,000 generations, sampled every 500th generation (1,000 total samples), and discarded the first 250 samples (25%) as burn-in, leaving 750 post-burn-in trees for the consensus.

The MrBayes settings appended to the NEXUS file are shown below. The key parameters are nst=6 (GTR, matching the ML model), rates=gamma (site rate variation), and ngen=500000 (total MCMC length). For larger datasets or to improve convergence, increase ngen to 2,000,000 or more and adjust burnin proportionally. 

begin mrbayes;
  set autoclose=yes;
  outgroup Bull_Shark_1;
  lset nst=6 rates=gamma;
  mcmc ngen=500000 samplefreq=500 printfreq=1000;
  sumt burnin=250;
  sump burnin=250;
end;

Running MrBayes

After the first R run writes the MrBayes block, switch to a terminal window, navigate into the R/MrBayes/ folder, and run: 

cd R/MrBayes
mb gulf_fish_COI_aligned.nex

MrBayes will print a progress table to the console every 1,000 generations and takes roughly 5–10 minutes for this 24-taxon dataset. When it finishes, it writes several output files into R/MrBayes/, including gulf_fish_COI_aligned.nex.con.tre (the consensus tree) and gulf_fish_COI_aligned.nex.p (the parameter log). Return to R and run 03_tree_Bayesian.R a second time — it will detect the output files and produce out/trees/tree_Bayesian.pdf.

Checking MCMC convergence

Before trusting the Bayesian tree, it's worth verifying that the MCMC actually converged. The free program Tracer (available at beast.community/tracer) reads the .p parameter log file and visualizes how each parameter evolved over the run. A well-converged run shows a "fuzzy caterpillar" trace — a rapidly fluctuating line with no visible trend after the burn-in — and reports an Effective Sample Size (ESS) above 200 for all parameters, especially the log-likelihood. If ESS values are low or the trace shows a drift, the chain hasn't mixed well, and you should re-run with a larger ngen.

Posterior probability (PP) is interpreted differently from ML bootstrap. A PP of 0.95 at a node means that 95% of post-burn-in sampled trees contained that exact clade — a direct probability statement about clade reality given the data and model. PP values tend to be numerically higher than ML bootstrap for the same node, and the two are not directly comparable. Use them as complementary lines of evidence: if both ML bootstrap and Bayesian PP are high, the result is robust; if PP is high but bootstrap is moderate, treat the node with some caution.

Extending This Analysis

Adding your own species

The pipeline is designed to be easy to extend. To add a new species, search NCBI Nucleotide for its COI sequences using the query format described in Section 2, download two or three sequences in FASTA format, give each one a clean header (e.g., >Spotted_Seatrout_1), and append the sequences to FASTA/gulf_fish_COI_combined.fasta. Then re-run the scripts from the beginning: 00_prepare_alignment.R first to re-align and regenerate the shared objects, then whichever tree scripts you want to update. If the new species belongs to a family not already in the dataset, add it to the family_colors vector in 00_prepare_alignment.R with a new hex color before running.

Using different genes

COI is the default barcode gene, but other mitochondrial markers work well for fish phylogenetics and may be better suited to specific questions. The 16S rRNA gene is a popular alternative — it is more conserved than COI, which makes it useful for resolving deeper divergences (across orders or classes) where COI may be too variable for reliable alignment. Cytochrome b (cytb) is another widely used mitochondrial marker with a large existing database in GenBank. Multi-gene analyses, which concatenate two or more markers into a single alignment, are generally more powerful than single-gene approaches because they provide more informative characters and reduce the chance that any one gene's evolutionary idiosyncrasies drive the result. The tradeoff is that multi-gene analyses require sequences from the same individuals — mixing accessions from different specimens across genes is not valid unless they come from verified voucher specimens.
